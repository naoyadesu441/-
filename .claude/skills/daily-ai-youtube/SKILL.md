# daily-ai-youtube — 海外AIニュース YouTube動画台本 自動生成

Stage 1（GitHub Actions）が毎朝 `news/YYYY-MM-DD.md` に「日本語要約＋裏取りステータス＋元リンク」を出力する。このスキルはそれを入力に、なおや式の動画台本を作って `scripts/youtube/YYYY-MM-DD.md` に書き出し＋Notion「YouTube動画台本」DBに投入する。

## 手順

1. **当日のリサーチを読む**
   - リポジトリ直下の `news/` から当日（JST）の `YYYY-MM-DD.md` を読む。
   - 無ければ `news/` 内の最新ファイルを使い、「最新分で代用した」旨を後述の出力末尾に注記する。

2. **台本にするネタを選ぶ（2〜4本）**
   - **裏取りステータスを最優先**で判断する:
     - 🟢一次確認済 / 🟡二次 を優先採用。
     - 🔴未確認は原則除外。どうしても扱う価値がある場合のみ採用し、台本中に必ず「※これは未確認情報です」と明示する。
   - 視聴者（AIを仕事で使いたい一般〜中級者）にとっての「実用性・驚き・話題性」で2〜4本に絞る。
   - 同じ話題の重複は1本にまとめる。

3. **台本を書く**
   - `台本原則.md`（同フォルダ）を必ず読み、その構成・トーン・尺・禁止事項に従う。
   - 各ネタを **共感フック → 具体数字 → 解決プロセス → 「実はこんなに簡単」 → CTA** の型で日本語の話し言葉にする。
   - 台本の各セクション末尾に、根拠として「元リンク」と「裏取りステータス」を残す（撮影者が確認できるように）。

4. **書き出す**
   - `scripts/youtube/YYYY-MM-DD.md`（JSTの当日日付）に保存する。
   - フォーマットは `台本原則.md` の「出力テンプレート」に従う。

5. **コミット & プッシュ**
   - `git add scripts/youtube/` → `git commit -m "youtube: YYYY-MM-DD AIニュース動画台本"` → `git push`。
   - ブランチは Routine の設定に従う（通常 main、またはレビュー運用なら作業ブランチ＋PR）。

6. **Notion「YouTube動画台本」DBを確認 or 作成**
   - Notion MCPで「YouTube動画台本」を検索する（Threads用の「AIニュース台本作成」とは**別DB**）。
   - **見つからない場合は以下のスキーマで新規作成する**:
     ```sql
     CREATE TABLE (
       "名前" TITLE,
       "番号" NUMBER,
       "ステータス" SELECT('ドラフト':gray, '撮影待ち':yellow, '公開済み':green, 'エラー':red),
       "公開日" DATE,
       "動画タイトル案" RICH_TEXT,
       "冒頭フック" RICH_TEXT,
       "本編" RICH_TEXT,
       "まとめ" RICH_TEXT,
       "ネタ数" NUMBER,
       "字数" NUMBER,
       "ネタの出典" RICH_TEXT,
       "裏取り" SELECT('一次確認済':green, '二次':yellow, '未確認':red),
       "修正メモ" RICH_TEXT,
       "再生数" NUMBER,
       "高評価数" NUMBER,
       "コメント数" NUMBER,
       "動画URL" URL,
       "サムネURL" RICH_TEXT,
       "エラー" RICH_TEXT
     )
     ```
   - 作成に失敗してもスキル全体は失敗にしない（ログして続行）。

7. **Notion「YouTube動画台本」DBに台本を投入**
   - 手順6で特定（or作成）したDBに、当日の台本を**1行**追加する。
   - プロパティマッピング:
     - `名前` ← 動画タイトル案の1案目（無ければ一番強いネタの見出し）
     - `ステータス` ← 「ドラフト」
     - `公開日` ← 当日日付
     - `動画タイトル案` ← タイトル案3案をまとめたテキスト
     - `冒頭フック` ← 「冒頭フック」セクション全文
     - `本編` ← 「本編」セクション全文（ネタ①②③…をまとめて）
     - `まとめ` ← 「まとめ＆クロージング」セクション
     - `ネタ数` ← 扱ったネタの件数（2〜4）
     - `字数` ← 台本全体の文字数
     - `ネタの出典` ← 各ネタの元リンクURL（改行区切り）
     - `裏取り` ← 複数ネタのうち最も弱いステータス（🟢一次確認済 / 🟡二次 / 🔴未確認）
   - 投入に失敗してもスキル全体は失敗にしない（ログして続行）。

8. **（任意）通知**
   - 必要に応じて Discord に、Claudeセッション内の MCP（GitHub）で完了を通知する。指示が無ければ通知は省略してよい。

## 注意
- リサーチに無い事実を創作しない。数字・固有名詞はリサーチ（および元リンク）の範囲に収める。
- 裏取りできていない内容を断定口調で語らない。
- 出力は日本語。なおやのチャンネルのトーンに合わせる（`台本原則.md` 参照）。
