# なおや式 台本原則（AIニュース動画）

> このファイルは台本の「型・トーン・尺」の定義書。`/daily-ai-script` がこれを読んで台本を書く。
> 既存の「⚡ kent-threads-daily-cloud」の文体・型に合わせたい場合は、この内容をその原則で上書き・調整してよい（ユーザーが編集する想定）。

## 想定
- **媒体**: YouTube（数分〜10分の解説動画。冒頭フックが重要）。必要ならショート用に短縮版も作れる。
- **視聴者**: AIを仕事・副業・日常で活用したい一般〜中級者。専門用語は噛み砕く。
- **話者**: 「なおや」。親しみやすく、押し付けない。視聴者と同じ目線で驚き、一緒に試す感じ。

## 基本構成（1ネタあたり）
各ネタを次の5ステップで構成する:

1. **共感フック（つかみ）**
   - 視聴者の悩み・あるある・期待に寄り添う一文から入る。
   - 例:「『AIに任せたいけど、結局むずかしそう』って思ってませんか？」
2. **具体数字（インパクト）**
   - リサーチにある数字（性能◯%向上、価格◯円、◯倍速 等）で「すごさ」を一発で伝える。
   - 数字はリサーチ/元リンクの範囲内のみ。盛らない。
3. **解決プロセス（中身の説明）**
   - 何が新しく、どう使うと何が嬉しいのかを順序立てて。専門用語は一言で補足。
4. **「実はこんなに簡単」（ハードルを下げる）**
   - 視聴者が今日から試せる手順・きっかけに落とす。「難しそうに見えて、実はこれだけ」。
5. **CTA（行動喚起）**
   - 「概要欄のリンクから試してみてください」「気になった人はコメントで」「次回は◯◯を解説します」等、自然に1つ。

## 動画全体の流れ
- **冒頭フック（15〜30秒）**: 今日いちばんの話題で心をつかむ（一番強いネタ or 全体の要約）。
- **本編**: 選んだ2〜4ネタを、上記5ステップで1本ずつ。重要度が高い順。
- **まとめ＆CTA（クロージング）**: 今日のポイントを1〜2文で振り返り、チャンネル登録/次回予告。

## トーン・文体
- 話し言葉。一文は短く、テンポよく。
- 専門用語は使ったらすぐ平易に言い換える。
- 煽りすぎ・断定しすぎを避ける。特に裏取りが弱いネタは「〜のようです」「現時点では未確認ですが」とトーンを落とす。
- 誇張・捏造は禁止。数字と固有名詞はリサーチ＋元リンクの範囲内。

## 裏取りの扱い
- **🟢一次（公式発表・論文）が台本の主役**。Stage1のリサーチ(news/*.md)は一次確認済のみが入る運用のため、通常🟡二次・🔴未確認は現れない。安心して具体的に語ってよい。
- 万一🔴未確認が混じっていても台本に使わない（正しさが確認できないものは扱わない方針）。

## 出力テンプレート（`scripts/YYYY-MM-DD.md`）
```
# 【AIニュース台本】YYYY-MM-DD

## 動画タイトル案（3案）
1. …
2. …
3. …

## 冒頭フック（15〜30秒）
（話し言葉の原稿）

## 本編

### ① <ネタ見出し>　[🟢一次/🟡二次/🔴未確認]
**共感フック:** …
**具体数字:** …
**解説:** …
**実はこんなに簡単:** …
**CTA:** …
> 参考: 元リンク <URL> ／ 裏取り: <ステータス>（必要なら裏取りリンク <URL>）

### ② <ネタ見出し>　[…]
（同上）

## まとめ＆クロージング
（振り返り1〜2文 ＋ チャンネル登録/次回予告）

---
出典: news/YYYY-MM-DD.md（Stage1リサーチ）
```
