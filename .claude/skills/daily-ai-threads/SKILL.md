# daily-ai-threads — 海外AIニュース Threads投稿 台本自動生成

Stage 1（GitHub Actions）が毎朝 `news/YYYY-MM-DD.md` に「日本語要約＋裏取りステータス＋元リンク」を出力する。このスキルはそれを入力に、伸びるThreads投稿の構成で台本を作り、`scripts/threads/YYYY-MM-DD.md` に書き出し＋Notion DBに投入する。

## 手順

1. **当日のリサーチを読む**
   - リポジトリ直下の `news/` から当日（JST）の `YYYY-MM-DD.md` を読む。
   - 無ければ `news/` 内の最新ファイルを使い、「最新分で代用した」旨を出力末尾に注記する。

2. **台本原則を読む**
   - 同フォルダの `台本原則.md` を**必ず読み**、構成・トーン・チェックリスト・出力テンプレートに従う。

3. **投稿にするネタを選ぶ（2〜4本）**
   - **裏取りステータスを最優先**で判断する:
     - 🟢一次確認済 / 🟡二次 を優先採用。
     - 🔴未確認は原則除外。どうしても扱う価値がある場合のみ採用し、投稿中に必ず「※未確認情報」と明示する。
   - 読者（AIを仕事で使いたい一般〜中級者）にとっての「実用性・驚き・話題性」で2〜4本に絞る。
   - 同じ話題の重複は1本にまとめる。

4. **投稿を書く（`台本原則.md` を厳守）**
   - 各ネタを `台本原則.md` の型（ツリー/短文/コメ誘導/箇条書き）で構成する。
   - **1行目で主張・結論を言い切る**。0.2秒で意味が掴める、意外性がある1行目にする。
   - 本文＋コメント欄1〜5 の構成で書く。**コピペでそのまま投稿できる状態**にする。
   - 書き終えたら `台本原則.md` の **セルフチェックリスト10項目を全部確認**。満たしていなければ修正する。

5. **書き出す**
   - `scripts/threads/YYYY-MM-DD.md`（JSTの当日日付）に保存する。
   - フォーマットは `台本原則.md` の「出力テンプレート」に従う。

6. **コミット & プッシュ**
   - `git add scripts/threads/` → `git commit -m "threads: YYYY-MM-DD AIニュースThreads投稿"` → `git push`。

7. **Notion「AIニュース台本作成」DBに投入**
   - Notion MCP を使い、各投稿を1行ずつDBに追加する。
   - プロパティマッピング:
     - `名前` ← 1行目（主張）の要約
     - `ステータス` ← 「ドラフト」
     - `投稿日` ← 当日日付
     - `型` ← ツリー / 短文 / コメ誘導 / 箇条書き
     - `本文` ← 投稿本文
     - `コメント欄1`〜`コメント欄5` ← 各コメント欄のテキスト
     - `字数` ← 本文の文字数
     - `ネタの出典` ← 元リンクURL
     - `裏取り` ← 一次確認済 / 二次 / 未確認
   - DBはNotion MCPで「AIニュース台本作成」を検索して特定する。
   - 投入に失敗してもスキル全体は失敗にしない（ログして続行）。

8. **（任意）通知**
   - Discord に完了通知を送る指示があれば実施。無ければ省略。

## 注意
- リサーチに無い事実を創作しない。数字・固有名詞はリサーチ（および元リンク）の範囲に収める。
- 裏取りできていない内容を断定口調で語らない。
- 出力は日本語。コピペでそのまま投稿できるクオリティにする。
- `台本原則.md` のセルフチェックリスト10項目は**省略不可**。
